## Contexto
Este é o card inicial do projeto. Para suportar o pipeline de avaliação de entrevistas, precisamos de uma infraestrutura base persistente e um banco de dados relacional para gerenciar e auditar o estado de cada entrevista. A durabilidade do estado no PostgreSQL é essencial para mitigar perdas de jobs na fila Redis (que opera in-memory sem AOF por padrão nesta fase).

## Objetivo
Configurar o ambiente Docker local (PostgreSQL + Redis) e definir o modelo de dados em Python utilizando SQLAlchemy ou SQLModel, mapeando os estados e metadados de uma entrevista.

## Especificação Técnica
- **Infraestrutura (Docker):**
  - Criar um `docker-compose.yml` contendo:
    - **PostgreSQL:** imagem `postgres:15-alpine`, configurado com credenciais padrão seguras e volume persistente.
    - **Redis:** imagem `redis:7-alpine`, configurado sem persistência agressiva por enquanto (para uso de fila rápida).
- **Modelagem de Dados (Python):**
  - Configurar gerenciamento de ambiente usando `python-dotenv` e `pydantic-settings` para carregar as URIs do banco e Redis.
  - Criar o modelo `Interview` no PostgreSQL com os seguintes campos:
    - `id`: UUID (Primary Key, gerado automaticamente).
    - `recording_url`: String (caminho local ou URL do áudio da gravação).
    - `status`: Enum contendo os estados: `recebida`, `transcrevendo`, `diarizada`, `pontuando`, `aguardando_aprovacao`, `aprovada`, `rejeitada`.
    - `transcription_raw`: JSON ou Texto (transcrição bruta consolidada).
    - `diarization_raw`: JSON (segmentos e timestamps do pyannote).
    - `scorecard`: JSON (estrutura final retornada pela IA com as avaliações por competência).
    - `created_at`: DateTime.
    - `updated_at`: DateTime.
  - Implementar script simples de inicialização de tabelas (ou setup do Alembic para migrations).

## Critérios de Aceite
