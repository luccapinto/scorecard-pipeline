## Contexto
Para validar o pipeline de áudio (transcrição, diarização e scoring) localmente sem expor dados confidenciais ou sensíveis de candidatos reais, precisamos de um mecanismo gerador de dados simulados. Esse gerador deve produzir tanto arquivos de áudio realistas de entrevistas (com falas alternadas entre entrevistador e candidato e termos técnicos em inglês) quanto os insumos de texto correspondentes (requisitos da vaga, checklist e escala comportamental BARS).

## Objetivo
Criar um script em Python capaz de gerar dados sintéticos completos de teste: arquivos de texto contextuais (JSON) e arquivos de áudio multi-voz simulando entrevistas reais com termos em português e inglês (code-switching).

## Especificação Técnica
- **Geração de Textos de Contexto:**
  - Criar estruturas e scripts Python para exportar em JSON:
    - `JobDescription`: Descrição da vaga (ex: Desenvolvedor Python Pleno).
    - `CompetencyFramework`: Lista de competências com âncoras de comportamento por nível (Escala BARS, do inglês *Behaviorally Anchored Rating Scales*), atribuindo descrições para pontuações de 1 a 5.
    - `EvaluationChecklist`: Lista de tópicos específicos a serem validados durante a entrevista.
- **Geração de Diálogo Sintético:**
  - Implementar um script que crie o roteiro da entrevista em formato de diálogo estruturado: `[{"speaker": "Entrevistador", "text": "..."}, {"speaker": "Candidato", "text": "..."}]`.
  - O roteiro deve incluir code-switching comum do ecossistema de TI no Brasil (ex: "deployar", "pull request", "build da pipeline", "refactoring").
- **Geração de Áudio (TTS):**
  - Utilizar uma biblioteca de Text-to-Speech (como `edge-tts` ou similar que ofereça vozes gratuitas em português brasileiro de boa qualidade).
  - Selecionar vozes distintas (pelo menos duas: ex. voz masculina e feminina do Azure via `edge-tts`) para ler as falas do entrevistador e do candidato.
  - Mesclar as falas geradas em um único arquivo de áudio (`.wav` ou `.mp3`) mantendo a ordem sequencial e simulando uma conversa natural.
  - Salvar as saídas estruturadas em `data/synthetic/`.

## Critérios de Aceite
- [ ] Script CLI disponível para execução (ex: `python -m scripts.generate_synthetic`).
- [ ] O script gera com sucesso arquivos de contexto (`job_*.json`, `competency_*.json`, `checklist_*.json`) e o arquivo de áudio correspondente (`interview_*.wav`).
- [ ] O áudio gerado possui vozes claramente distintas para cada speaker (audível e analisável pelo Pyannote).
- [ ] O conteúdo gerado cobre code-switching PT-EN com terminologias de engenharia de software.

### Testes Automatizados
- [ ] Criar testes automatizados (`pytest`) para validar a integridade dos arquivos JSON gerados contra schemas Pydantic.
- [ ] Validar que o gerador de áudio cria um arquivo válido contendo frames de áudio legíveis por bibliotecas padrão (como `soundfile` ou `wave`).
- [ ] Rodar a suíte inteira de testes e verificar se está 100% verde.
