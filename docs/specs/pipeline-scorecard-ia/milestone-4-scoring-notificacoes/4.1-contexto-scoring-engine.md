# Card 4.1 — Agregador Contexto & Scoring Engine

## Contexto
O valor principal da pipeline de scorecard automatizado é a geração de avaliações objetivas e auditáveis. Para isso, o motor de scoring não pode depender de RAG (que pode omitir trechos da entrevista), optando por lookup determinístico dos arquivos contextuais da vaga. A avaliação das competências deve seguir a escala BARS (*Behaviorally Anchored Rating Scales*) e, crucialmente, cada nota deve vir acompanhada por uma citação literal da transcrição como evidência. Para combater alucinações, o sistema deve rodar uma rotina pós-LLM validando a veracidade de cada citação.

## Objetivo
Implementar a agregação determinística do contexto da vaga, a chamada do OpenRouter estruturada com Pydantic e o algoritmo de pós-processamento de validação de citações literais da transcrição.

## Especificação Técnica
- **Agregador de Contexto (`ContextAggregator`):**
  - Implementar leitura dos arquivos locais baseando-se no `job_id` da vaga.
  - Carregar os arquivos JSON `job_*.json`, `competency_*.json` e `checklist_*.json`.
- **Motor de Scoring (`ScoringEngine`):**
  - Configurar cliente OpenRouter (via HTTP client ou biblioteca OpenAI apontada para `https://openrouter.ai/api/v1`).
  - Definir o schema Pydantic de saída esperado:
    ```python
    class CompetencyEvaluation(BaseModel):
        competency_name: str
        score: int  # Escala 1 a 5 (BARS)
        justification: str
        evidence_quote: str  # Citação exata falada na entrevista
        evidence_verified: Optional[bool] = None

    class ScorecardOutput(BaseModel):
        candidate_name: str
        overall_recommendation: str  # ex: Aprovado, Rejeitado, Próxima Etapa
        evaluations: List[CompetencyEvaluation]
    ```
  - Montar prompt contextualizado instruindo a IA a avaliar o candidato nos termos da vaga e a justificar os pontos com citações literais exatas do texto da transcrição.
- **Validador de Evidências:**
  - Desenvolver algoritmo de verificação que pesquisa a substring `evidence_quote` dentro da transcrição bruta consolidada (`transcription_raw`).
  - Para evitar falso-negativos por diferenças pequenas de espaçamento ou pontuação, normalizar tanto a citação quanto a transcrição (remover pontuações, converter para minúsculas) antes da validação.
  - Definir o campo `evidence_verified` como `True` se a citação literal constar na transcrição e `False` caso contrário.

## Critérios de Aceite
- [ ] O `ContextAggregator` lê os arquivos locais da vaga dinamicamente e monta o payload.
- [ ] O `ScoringEngine` consome a API do OpenRouter e realiza o parse correto do schema Pydantic.
- [ ] A lógica de validação de citações de evidência funciona:
  - Marca `evidence_verified = True` para citações existentes.
  - Marca `evidence_verified = False` e gera um log de warning para citações alucinadas pelo LLM.
- [ ] O scorecard estruturado é persistido no banco de dados da entrevista no PostgreSQL.
