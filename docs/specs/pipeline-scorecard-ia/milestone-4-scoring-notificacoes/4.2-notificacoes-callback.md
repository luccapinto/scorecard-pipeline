# Card 4.2 — Notificações e Callback Humano

## Contexto
O processo de avaliação estruturada exige um Humano no Loop (HITL) para auditar e chancelar a decisão sugerida pelo pipeline de IA. O scorecard gerado deve ser enviado a interfaces interativas (como Slack, e-mail ou webhook externo). A partir dessas interfaces, o avaliador humano deve conseguir aprovar ou rejeitar o scorecard, disparando um sinal de volta (callback) que consolida o estado final da entrevista no banco de dados.

## Objetivo
Implementar a interface de notificação plugável, os adaptadores para Slack (Block Kit) e Webhook genérico, além do endpoint de callback no FastAPI para registrar a decisão humana e finalizar a máquina de estados.

## Especificação Técnica
- **Interface de Notificação (`BaseNotification`):**
  - Definir a interface `BaseNotification` com o método `notify_scorecard(interview_id: UUID, scorecard: dict) -> None`.
- **Adaptadores de Notificação:**
  - **Slack (`SlackNotification`):**
    - Montar layout interativo utilizando Slack Block Kit contendo:
      - Cabeçalho com o nome do candidato e cargo.
      - Sumário com a recomendação geral e notas por competência.
      - Trechos de evidência marcados com o status de validação (`[OK]` ou `[ALERTA: Alucinação detectada]`).
      - Botões interativos para "Aprovar" e "Rejeitar", apontando para a URL do callback.
  - **Webhook Genérico (`WebhookNotification`):**
    - Enviar payload HTTP POST JSON completo do scorecard para uma URL externa cadastrada nas variáveis de ambiente.
- **Endpoint de Callback (`POST /interviews/{interview_id}/action`):**
  - Payload esperado: `{ "action": "approve" | "reject" }`.
  - Regras de negócio do callback:
    1. Buscar a entrevista pelo ID no banco.
    2. Validar se o status atual da entrevista é `aguardando_aprovacao`. Se for diferente (ex: já aprovado anteriormente ou ainda processando), retornar HTTP 400 (Bad Request).
    3. Atualizar o status do banco para `aprovada` ou `rejeitada` de acordo com a ação recebida.
    4. Persistir a data e hora da decisão final (`updated_at`).

## Critérios de Aceite
- [ ] O pipeline envia notificações contendo os dados do scorecard de forma estruturada.
- [ ] O endpoint de callback FastAPI `/interviews/{interview_id}/action` aceita requisições HTTP válidas e atualiza o estado correspondente no Postgres.
- [ ] Requisições de callback para entrevistas que não estejam no estado `aguardando_aprovacao` falham de forma segura com HTTP 400.
- [ ] A aplicação suporta a ativação dinâmica de múltiplos canais de notificação através das variáveis de ambiente.
