## Contexto
O code-switching (mistura de português e termos técnicos em inglês) é frequente em entrevistas de tecnologia. Para decidir entre o processamento local gratuito (WhisperX) e a API da OpenAI (paga, porém mais robusta a ruídos e termos específicos), precisamos mensurar empiricamente a qualidade de transcrição de ambos os motores utilizando o Word Error Rate (WER) em cima dos nossos dados sintéticos (dos quais temos o texto de gabarito exato).

## Objetivo
Criar um script em Python para rodar benchmarks de transcrição, calculando a métrica WER (Word Error Rate) e gerando um relatório analítico comparativo formatado em Markdown.

## Especificação Técnica
- **Biblioteca de Métricas:**
  - Utilizar a biblioteca `jiwer` (ou implementar algoritmo Levenshtein robusto) para calcular a taxa de erro de palavra (WER).
- **Processamento de Normalização de Texto:**
  - Implementar uma função de limpeza de texto para garantir comparações justas: converter para minúsculas, remover pontuação (`.`, `,`, `?`, `!`, `-`), remover espaçamentos extras e normalizar acentos se necessário.
- **Fluxo do Script de Benchmark (`scripts/run_benchmark.py`):**
  - Iterar sobre as amostras sintéticas de áudio geradas no Milestone 1.
  - Obter o texto de referência do diálogo sintético correspondente.
  - Executar a transcrição via `LocalTranscription` (WhisperX) e calcular o WER local.
  - Executar a transcrição via `OpenAITranscription` (OpenAI API) e calcular o WER da API.
  - Gerar o relatório formatado em markdown com:
    - Tabela comparativa (Arquivo de Teste, WER Local, WER OpenAI API).
    - Média geral de WER para cada provider.
    - Análise qualitativa automática: trechos onde ocorreram os maiores desvios, com destaque especial em palavras de code-switching (como "deploy", "merge", etc.).
  - Gravar o arquivo final em `docs/reports/benchmark_wer_report.md`.

## Critérios de Aceite
- [ ] O script de benchmark é executável via linha de comando.
- [ ] O cálculo do WER é feito individualmente por amostra e uma média é sumarizada.
- [ ] Um relatório Markdown é gerado e atualizado contendo tabelas e análises dos erros de transcrição.
- [ ] A normalização de texto remove formatações arbitrárias para focar no conteúdo das palavras.

### Testes Automatizados
- [ ] Criar testes unitários para a função de normalização de texto.
- [ ] Validar via teste unitário que a fórmula do WER está correta comparando uma transcrição de teste contra uma referência controlada com erros conhecidos de omissão, inserção e substituição.
- [ ] Rodar a suíte inteira de testes e verificar se está 100% verde.
