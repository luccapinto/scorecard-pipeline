# Card 3.1 — Motor Transcrição e Diarização Pluggable

Tag: 🤖 Execução Autônoma

## Contexto
O processamento de áudio exige a combinação de duas técnicas: Transcrição (conversão de fala em texto) e Diarização (quem falou o quê). Como os requisitos de infraestrutura variam (custo local de GPU vs. custo de API/latência), o pipeline precisa aceitar provedores plugáveis de transcrição (WhisperX local vs. OpenAI API na nuvem). Visto que a API da OpenAI não oferece diarização nativa, precisamos implementar um algoritmo robusto de alinhamento baseado em sobreposição de timestamps para cruzar o resultado do Whisper com os segmentos identificados pelo `pyannote.audio`.

## Objetivo
Implementar a interface de transcrição plugável, os drivers para transcrição local (WhisperX) e API (OpenAI), a integração do `pyannote.audio` para diarização e a lógica de mesclagem (merge) de transcrição/diarização.

## Especificação Técnica
- **Interface Base (`AudioProcessor`):**
  - Definir a interface `BaseTranscription` com o método `transcribe(audio_path: Path) -> List[Dict]`.
- **Driver Local (`WhisperX`):**
  - Implementar `LocalTranscription` utilizando a biblioteca `WhisperX` ou `faster-whisper`.
  - Integrar com o modelo Whisper pré-configurado.
- **Driver de API (`OpenAI API`):**
  - Implementar `OpenAITranscription` utilizando a biblioteca `openai` oficial para chamar o endpoint de Transcriptions (`whisper-1`).
- **Módulo de Diarização (`pyannote.audio`):**
  - Criar o módulo `Diarizer` encapsulando o pipeline do Pyannote (`pyannote/speaker-diarization-3.1`).
  - Ler o `HF_TOKEN` do ambiente para autenticação com o Hugging Face.
- **Lógica de Alinhamento e Mesclagem (Merge):**
  - Implementar o alinhamento de falas (necessário principalmente para o driver OpenAI):
    - Receber a lista de segmentos do Pyannote: `[{"start": 1.2, "end": 4.5, "speaker": "SPEAKER_01"}, ...]`.
    - Receber a transcrição da OpenAI contendo palavras/frases e seus respectivos timestamps `[{"text": "Olá bom dia", "start": 1.3, "end": 3.0}, ...]`.
    - Atribuir o falante (`speaker`) a cada segmento transcrito com base no overlap temporal máximo (maior intersecção de tempo).
  - Retornar uma estrutura de dados consolidada e ordenada temporalmente:
    ```json
    [
      {
        "speaker": "SPEAKER_01",
        "text": "Olá bom dia, gostaria de iniciar a entrevista.",
        "start": 1.2,
        "end": 4.5
      }
    ]
    ```

## Critérios de Aceite
- [x] Interface comum implementada de forma a permitir a troca de providers dinamicamente via variáveis de ambiente (`TRANSCRIPTION_PROVIDER=local` ou `openai`).
- [x] O pipeline de diarização do Pyannote roda isoladamente e retorna os segmentos com identificação de speaker.
- [x] O algoritmo de mesclagem (merge) resolve corretamente o speaker para segmentos de texto com base no timestamp, mesmo com pequenas variações de sincronismo do Whisper.
- [x] Os resultados da transcrição e diarização são salvos nos respectivos campos `transcription_raw` e `diarization_raw` do PostgreSQL.
